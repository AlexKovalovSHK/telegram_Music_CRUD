## Принцип работы с документом

Каждый сценарий проверяется вручную или автоматически после завершения соответствующей фазы.
Колонка «Фаза» указывает, после какой фазы из `04_PLN_implementation.md` сценарий должен проходить.

## 1. Backend: API

| ID | Сценарий | Входные данные | Ожидаемый результат | Фаза |
|----|----------|---------------|-------------------|------|
| B-01 | Получить пустой список | `GET /pieces` | `200 []` | 3 |
| B-02 | Создать произведение | `POST /pieces` с валидным телом | `201 { id, deNum, deName, ruNum, ruName }` | 3 |
| B-03 | Получить список после создания | `GET /pieces` | `200 [{ id: 1, ... }]` | 3 |
| B-04 | Получить одно произведение | `GET /pieces/1` | `200 { id: 1, ... }` | 3 |
| B-05 | Обновить произведение | `PUT /pieces/1` с `{ ruName: "Новое" }` | `200 { id: 1, ruName: "Новое", ... }` | 3 |
| B-06 | Удалить произведение | `DELETE /pieces/1` | `200 { id: 1 }` | 3 |
| B-07 | Получить удалённое произведение | `GET /pieces/1` после удаления | `404 { statusCode: 404, message: "...", error: "Not Found" }` | 3 |

| B-10 | Обновить несуществующее | `PUT /pieces/999` | `404` | 3 |
| B-11 | Удалить несуществующее | `DELETE /pieces/999` | `404` | 3 |
